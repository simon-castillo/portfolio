Project teamRight to work

BriefMap the user journey of proving and checking right to work in the UK

Prototyping solutions

ProjectRight to work

SkillsPrototyping with the GOV.UK kit, HTML, CSS & Javascript

Prototyping in code allows for realistic feedback from users and unifies the team and stakeholders' vision of the product. I use HTML, CSS and Javascript to prototype new interactions, which are then tested with users to find out how we can improve future iterations.

Simplifying interactions

ProjectAn internal reporting platform

BriefAllow users to sanitise text imported from another system

SkillsInterpreting user research, low fidelity prototyping

In user research sessions, we noticed that people’s existing method for cleaning up imported text was to paste the content into Word, then edit and paste back into the platform afterwards.

In response, I created clickable prototypes to test more efficient solutions. As these interactions were very different to others on the platform, I kept fidelity low, focusing on validating the rough idea. They have since been recognised as a pattern that can increase efficiency across the whole platform.

Supporting the design profession

SkillsVisual design and communication

The stronger our design community is, the more we can champion user centred design around the Home Office. To support this, I have designed training resources for workshops and presented work at cross government meetups to learn and share with other designers.

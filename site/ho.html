Mapping a service

Project teamRight to work

BriefMap the broad user journey of proving and checking right to work in the UK

These service maps were created to give a wider view of the user journeys for both job seekers and employers. They are grounded in user research and allow the project team to find areas of the service to improve, while being a useful internal communication tool.

Cleaning up imported text

ProjectAn internal reporting platform

BriefDesign a way for users to sanitise reports that have been imported automatically from another reporting system.

In user research sessions, we noticed that people’s existing method for cleaning up imported text was to paste the content into Word, then edit and paste back into the platform afterwards. We wanted to let them sanitise the report more efficiently, in one screen.

So, at the end of the import flow, when the user lands on the editable report, I added a sidebar that maintains the original data - uneditable - for reference. The user can open this, compare directly to their new report and add or remove data as needed.

Dashboard designs

ProjectAn internal reporting platform

BriefDesign dashboards for viewing platform usage metrics

The task here was to iterate on existing, ‘blue-sky’ dashboard designs that had not yet been built, and update them to display the latest available dataset.

Grouping and hierarchy were especially important. The user needs to perceive and understand the information at a glance, so text was kept to a minimum and data was visualised wherever possible.
